Capabilities

What it does

Three production-shaped pipelines, all running locally on Apple Silicon.

🔍
Hybrid RAG

Reciprocal Rank Fusion of BM25 (lexical) and dense embeddings (semantic), optionally reranked by a cross-encoder. Catches exact-match identifiers (FD001, sensor_11) and conceptual queries in one pass.

🤖
Agent with Python tool calling

A ReAct agent that can call query_cmapss on a pandas DataFrame. Closed DSL — 7 whitelisted operations, no arbitrary code execution, clean error messages on bad input. 5/5 tool invocations on the A/B test harness.

📊
RAGAS evaluation

Standard metrics — faithfulness, answer relevancy, context precision, context recall — on a deterministic 30-item Q&A dataset. Immutable snapshots per run give a time series across tuning iterations.

🛡️
EU AI Act aware

Transparency (Article 50), risk register, data governance, human oversight — all documented. No PII in the corpus. Architecture: minimal-risk + transparency obligations only.

🔬
A/B tested LLM choice

Qwen2.5-7B-Instruct (4-bit) over Mistral-7B-Instruct-v0.3 — same size, same RAM, but 5/5 vs 2/5 tool invocations on the ReAct harness. Decision driven by measurement, not preference.

🐛
Real bugs, real fixes

20 bugs caught and resolved before first real run — including withdrawn PyPI versions, tokenizers conflicts, Pydantic v2 ClassVar issues, missing stop sequences, and a CP1252 encoding trap in the NASA readme.

Architecture

Three pipelines, one shared index

Ingestion builds the index, query serves users, evaluation measures quality. All share the same ChromaDB collection.

Ingestion (one-shot)

NASA CMAPSS (.txt) Schaeffler + SKF (.pdf) │ │ ▼ ▼ cmapss_loader (pandas) pdf_loader (PyMuPDF) │ │ └──────────────┬───────────────────┘ ▼ _dataframe_to_text (markdown block per subset) │ ▼ recursive_split (500 tok / 50 overlap) │ ▼ Embedder.embed (bge-small + MPS, 384-dim) │ ▼ chunks.jsonl (audit trail) VectorStore.upsert (HTTP :8001)

Query (per request)

User │ ▼ HTTP POST Streamlit UI :8501 │ ▼ POST /query {question, top_k} FastAPI :8000 │ ▼ RAGChain.query() HybridRetriever.retrieve() (over-fetch ×3 if rerank enabled) │ ├── dense → ChromaDB.cosine └── BM25 → in-memory index │ ▼ RRF Reranker.rerank() (if enabled) │ ▼ top_k RAGChain._format_context │ ▼ prompt + context + question │ ▼ MLXChatModel Qwen2.5-7B-Instruct (4-bit) + Metal │ ▼ RAGResponse{answer, sources}

Evaluation (per release)

eval_dataset.jsonl (30 Q&A, seed=42) │ ▼ RAGChain.query() ×30 │ ▼ ragas.evaluate(metrics=[ faithfulness, answer fidèle au contexte answer_relevancy, answer pertinente à la question context_precision, les bons chunks ont été retrievés context_recall, tous les chunks nécessaires retrieved ]) │ ▼ reports/eval_<UTC>.json (immutable, jamais overwritten)

Decision by measurement

A/B test: Qwen2.5 vs Mistral

Same harness, same 5 quantitative CMAPSS questions, only the model changed. Before the A/B, two real code bugs (stop sequences, return_intermediate_steps) were fixed — otherwise the test would have masked bugs behind a model change.

Model Quant. Tool correctly invoked Answer correct Verdict
Mistral-7B-Instruct-v0.3 4-bit 2 / 5 1 / 3 Rejected
Qwen2.5-7B-Instruct 4-bit 5 / 5 2-3 / 3 Selected
Why Mistral failed: it wrapped the tool name in markdown backticks (`query_cmapss`), breaking LangChain's exact-match lookup, and could loop until iteration limit. Qwen2.5 produced clean invocations in 5 out of 5 cases, with no overhead (same 7B size, same RAM footprint).

Quickstart

From zero to demo in ~35 min

Five commands, then ask a question in the UI. Tested on MacBook Pro M5 Pro. 

# 1. One-time setup (5 min)
git clone https://github.com/PDUCLOS/industrial-knowledge-copilot
cd industrial-knowledge-copilot
make setup              # venv + ~24 packages
make pull-models        # Qwen2.5-7B + bge-small (~5 GB, ~30 min)
make data               # NASA CMAPSS (~12 MB)
make chroma-up          # ChromaDB on :8001

# 2. Build the index (5 min on first run)
make ingest
make eval-dataset

# 3. Run the demo (two terminals)
make api                # FastAPI on :8000
make ui                 # Streamlit on :8501

# 4. Open http://localhost:8501 and ask:
"How many turbofan engines are in the FD001 training set?"
10 demo questions ready for the pitch are in scripts/demo_questions.md — cover factual, reasoning, multi-hop (agent), Schaeffler/SKF, and out-of-scope categories.
